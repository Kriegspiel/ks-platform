# Slice 810 - Regression Gate Hardening

## Objective

Create a deterministic, automation-first regression baseline that reliably blocks regressions before merge.

## Scope

- Backend unit/integration regression command standardization
- Frontend unit/component regression command standardization
- Cross-surface integration smoke execution in CI-compatible form
- Coverage and flaky-test policy formalization

## Deliverables

- Updated test command wrappers/scripts if missing
- Regression matrix for core product journeys
- CI required-check mapping and gate thresholds
