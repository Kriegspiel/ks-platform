# Slice 960 - Content Repository Organization and Operational Hygiene

## Objective

Define and enforce a maintainable `Kriegspiel/content` repository structure so editorial work scales safely with deterministic build behavior.

## Scope

- Repository folder architecture and naming conventions
- Content ownership and CODEOWNERS mappings
- Lifecycle state conventions (draft/review/published/archived)
- Automated structure and policy validation in CI

## Deliverables

- Content repo organization specification
- Ownership and review policy definitions
- CI checks for folder, filename, and lifecycle policy compliance
